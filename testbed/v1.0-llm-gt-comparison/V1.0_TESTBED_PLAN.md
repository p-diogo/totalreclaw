# TotalReclaw v1.0 Testbed and Benchmark Plan

## Context

**Problem**: Before implementing TotalReclaw (a zero-knowledge E2EE memory vault for AI agents), we need to validate that its search accuracy and performance are competitive with state-of-the-art solutions (OpenClaw built-in memory, QMD/QMath hybrid).

**What We Have**:
- v0.2 spec: 2-pass search (remote KNN + local BM25/RRF)
- v0.5 spec: 3-pass search (adds LLM reranking + multi-variant blind indices)
- Previous v0.X evaluations completed (Feb 18-19, 2026)
- Existing reusable assets: 150 queries, 1,500 memories, LLM ground truth, baseline scripts, E2EE evaluation with real AES-GCM encryption

**What We Need**:
- Comprehensive v1.0 benchmark with clean folder structure
- LLM rerank compute bottleneck testing (v0.5 specific - missing from previous tests)
- Apple-to-apple comparison across all algorithms
- Final go/no-go decision report

## Implementation Steps

### Step 1: Setup & Cleanup
- Create `testbed/config/api_keys.env` template file
- User adds OpenRouter API key
- Clean up `v1.0-llm-gt-comparison/data/processed/`:
  - Keep only `memories_1500_final.json` (rename to `memories.json`)
  - Remove intermediate files (memories_v2.json, memories_add*.json, etc.)
- Create `v1.0-llm-gt-comparison/scripts/`, `results/`, `reports/` folders

### Step 2: Create Baseline Runner (`01_run_baselines.py`)
- Import and run all 4 baseline algorithms
- Use same data files across all
- Output unified `results/baselines.json`

### Step 3: Create v0.2 Runner (`02_run_totalreclaw_v02.py`)
- Import from `totalreclaw_v02_eval.py`
- Add detailed timing breakdown:
  - Pass 1: KNN query latency
  - Pass 2: Decryption + BM25 + RRF timing
- Output `results/totalreclaw_v02.json`

### Step 4: Create v0.5 Runner (`03_run_totalreclaw_v05.py`)
- Modify `totalreclaw_v05_eval.py` to use real LLM via OpenRouter (`arcee-ai/trinity-large-preview:free`)
- Run two modes: without LLM (S6) and with LLM (S7)
- Capture LLM call timing separately
- Output `results/totalreclaw_v05.json`

### Step 5: Create LLM Rerank Benchmark (`04_benchmark_llm_rerank.py`)
- Isolated test of LLM reranking using `arcee-ai/trinity-large-preview:free`
- Test with varying candidate counts: 10, 20, 30, 50
- Measure: per-query latency, token count, API call time
- Output `results/llm_rerank_benchmark.json`

### Step 6: Create Report Generator (`05_generate_report.py`)
- Load all result files
- Generate comparison tables
- Create `reports/EVALUATION_REPORT.md` with:
  - Executive summary + go/no-go
  - Algorithm comparison (accuracy + latency)
  - E2EE overhead analysis
  - LLM rerank bottleneck analysis
  - Recommendations

### Step 7: Create Master Runner (`run_all.py`)
- Execute S1-S8 in order
- Generate final report
